# What is an Algorithm?

## What is an Algorithm?

An algorithm is a set of steps to solve a problem. It is the *action* — not how data is stored. It can use a data structure, but it doesn't have to. Example: checking if a number is prime needs no data structure at all.

---

## What Makes a Valid Algorithm

- **Input** — clear data it takes in
- **Output** — a clear result it gives back
- **Finiteness** — it must stop after a limited number of steps
- **Definiteness** — each step must be clear, not confusing
- **Effectiveness** — each step must be simple enough to actually do

---

## Algorithm vs Program

- **Algorithm** = the plan (not tied to any language)
- **Program** = the plan written in real code (Python, Java, etc.)

The same algorithm can be written in many different languages.

---

## How We Judge an Algorithm

1. **Correctness** — does it give the right answer every time?
2. **Efficiency** — how fast is it, and how much memory does it use? (This is Big-O.)

---

## Common Algorithm Types

- **Brute Force** — try every option, no shortcuts
- **Divide and Conquer** — split the problem into smaller parts, solve each, then combine
- **Greedy** — pick the best option at each step, one step at a time
- **Dynamic Programming** — solve repeated smaller problems once, save the results
- **Backtracking** — try a solution step by step, go back if it fails

---

## Summary

An algorithm is a clear, limited set of steps to solve a problem. We judge it by two things: does it work correctly, and how fast/efficient is it. Most problems use one of these patterns: brute force, divide and conquer, greedy, dynamic programming, or backtracking.
