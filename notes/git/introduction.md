# Git Introduction

## What is Git?

Git is a **distributed version control system** used to track changes in software projects.

It helps developers:

* Track code history.
* Manage changes.
* Collaborate with others.
* Restore previous versions.

---

## Why Git?

Without version control, managing code changes becomes difficult.

Git provides:

* A history of changes.
* Safe experimentation.
* Team collaboration.

---

## Git Repository

A repository is a place where Git stores:

* Project files.
* Change history.
* Commits.

---

## Git Workflow

```text
Working Directory
        ↓
Staging Area
        ↓
Repository
        ↓
Remote Repository
```

* **Working Directory:** Files you modify.
* **Staging Area:** Changes prepared for commit.
* **Repository:** Saved project history.
* **Remote:** Shared repository (GitHub, GitLab, etc.).



## Summary

Git helps developers manage code versions, track changes, and collaborate efficiently.
