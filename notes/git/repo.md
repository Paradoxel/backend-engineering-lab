# Git Repository

## What is a Repository?

A **repository (repo)** is the place where Git stores your project, its history, and all tracked changes.

It contains:

* Project files.
* Commit history.
* Branches.
* Git metadata.

---

## Types of Repository

### Local Repository

Stored on your own machine.

This is where you develop your project.

---

### Remote Repository

Stored on a remote server such as:

* GitHub
* GitLab
* Bitbucket

Used for collaboration and backup.

---

## Repository Structure

A Git repository can be viewed as three main areas:

```text id="z2wnnz"
Working Directory
        ↓
Staging Area
        ↓
Local Repository
```

* **Working Directory** → Files you are currently editing.
* **Staging Area** → Changes prepared for the next commit.
* **Local Repository** → Permanent Git history.

---

## Why Repositories?

Repositories allow developers to:

* Organize projects.
* Track history.
* Collaborate with others.
* Restore previous versions.
* Manage multiple branches.

---

## Summary

A repository is the central place where Git manages a project's files, history, and version control information.
